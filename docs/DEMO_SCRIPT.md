# CustomerPulse AI demo script — under three minutes

Use the deployed URL in a fresh browser profile. No credentials are required. If prior demo state exists, open Data Imports as Administrator and choose Reset Demo.

## 0:00–0:20 — Problem and governed design

Open Overview.

Say: “CustomerPulse AI turns fragmented customer evidence into explainable retention and marketing work. AVO assists, while deterministic scoring, consent, role gates, human approval, and audit remain in control.”

Point out Synthetic Demo Data, the persisted Switch Role control, and the two guided demo buttons.

## 0:20–1:25 — Scenario A: evidence to governed action

1. Start Guided Demo · Scenario A, or open Conversations as Account Executive.
2. Select Maya Tan and run AVO Analysis. Show message evidence IDs, confidence, uncertainty, and the statement that risk is not a prediction.
3. Generate/open REC-001 in Recommendations. Show owner, deadline, editable human draft, approver, and evidence.
4. Submit for Approval. Read the success state: it went to Sales Manager and created ACT-021. Choose View Retention Action; point out the query ID and highlighted Pending Approval row.
5. Switch to Sales Manager, enter “Evidence and service policy checked,” then Approve. Emphasize that the action is Approved and Ready—not executed.
6. Switch to Account Executive and choose Execute Approved WhatsApp Action. No message is auto-sent; CustomerPulse records execution and asks for an outcome.
7. Enter a customer response and outcome, record it, then open Audit Reports to show requester, approver and executor events.

## 1:25–2:30 — Scenario C: insight to shared calendar

1. Switch to Marketing Manager and open Marketing Intelligence. Show MKT-003: what changed, affected customers, revenue at risk, drivers, supporting evidence, recommended response, what to avoid, expected outcome, confidence and uncertainty.
2. Choose Create campaign with AVO. Show the prefilled Brief and the seven real steps: Brief, Audience, Sources, Content, Review, Approval, Schedule.
3. Continue through Audience and Sources. Generate content from approved product catalogue/guidelines and note editable versions and factual validation.
4. In Review, confirm audience, consent, sources, content, claims and schedule.
5. In Approval, enter a reviewer comment, Submit for Approval, then Approve. State that self-approval and missing comments are blocked.
6. Continue to Schedule and choose Schedule Campaign. In no-key mode the Demo Publisher is visibly simulated.
7. Campaign Calendar opens with CAM-003 highlighted. Show one ScheduledPost per channel, date/time zone, provider, approver and publisher IDs.

## 2:30–2:55 — Management and reliability

Open Analytics and change the Tier filter. Show that filtered KPIs, management insights and the customer table update together. Point out observed/calculated/AVO/recommendation labels and the warning that correlation is not causation.

Close: “The same system works without credentials using deterministic AVO and publisher fallbacks. Optional OpenAI, Buffer and Supabase adapters are present but not described as live-verified without keys.”

## Presenter fallback

If live OpenAI is unavailable, continue with AVO Demo Analysis; that is the verified path. If Buffer is unavailable, use Demo Publisher; it creates real in-app ScheduledPost records but does not post externally. If an upload fails, read the in-page status and do not claim success—use the bundled file again after confirming the role and type.

## Phase 1 calculated demo (under three minutes)

1. In the header choose Demo Workspace and Reset Demo Data. Explain that Imported Workspace is preserved.
2. Open Maya Tan in Conversations and Run AVO Analysis. Point out validated message evidence, confidence, uncertainty, the new official churn calculation, and alert.
3. Open the recommendation, edit it, and submit. As Sales Manager request changes with explicit feedback. Return as Account Executive, Begin Revision, edit, and resubmit.
4. As Sales Manager approve. Return as Account Executive, Start Action, open the approved WhatsApp control, and Confirm Execution.
5. Record Mayas response. Show that it is a separate record and the state becomes Outcome Required. Select Complaint resolved, add notes, and record the outcome.
6. Open Customer 360/Analytics/Audit. Show calculated before/after score, revenue-at-risk change, updated alert, and the complete event chain.
7. Optionally open Omars Approved and Ready action and repeat the recovery path; the score decrease comes from the same engine, not a scenario constant.
